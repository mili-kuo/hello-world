Forge

An autonomous AI coding agent that plans, writes, tests, and ships code — right inside your repository.

What is Forge?

Forge is an AI-powered coding agent built into the Aine platform. You describe a feature, a bug fix, or any code change in plain English, and Forge takes care of the rest — reading your codebase, writing the code, running tests, committing, and pushing to your branch. It works like a careful senior engineer: it makes a plan, executes each step, verifies the result, and only declares done when everything is clean and working.

Key Features

🗺️

Transparent Planning

Forge shows its step-by-step plan before writing a single line of code, so you always know what's happening.

🤖

Autonomous Execution

It reads files, writes code, runs builds and tests, and iterates until the job is done — no hand-holding needed.

🔒

Sandboxed & Safe

Every session runs in an isolated container with restricted network access, keeping your environment secure.

📚

Living Knowledge Base

Forge records what it learns about your codebase in a wiki, so future sessions get smarter over time.

🌿

Git-Native

All work happens on a dedicated feature branch. Commits are incremental and descriptive, ready for review.

💬

Asks When Unsure

When a task is genuinely ambiguous, Forge pauses and asks you — rather than guessing and going in the wrong direction.

How It Works

  1. You describe what you want built or fixed in plain English.
  2. Forge explores the repository to understand the existing code and conventions.
  3. It creates a plan with 3–8 concrete steps and shows it to you in real time.
  4. It executes each step — writing code, running commands, and verifying results.
  5. All changes are committed with descriptive messages on a dedicated feature branch.
  6. You review the branch and merge when you're happy with the result.

Built for Real Codebases

Forge isn't a code-snippet generator. It operates directly inside your repository, respects your project's existing style, runs your actual test suite, and treats failing tests as blockers rather than noise. Whether your project is a simple static site or a multi-package monorepo, Forge adapts to the tools and conventions already in place.
